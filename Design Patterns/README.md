# Design Patterns (LLD)

This repository contains implementations and explanations of classic **Design Patterns** used in Low-Level Design (LLD).

Design patterns are reusable solutions to commonly occurring design problems in software design. They help improve:
- Code reusability
- Maintainability
- Scalability
- Flexibility

---

## Categories of Design Patterns

Design patterns are broadly classified into three categories:

---

## 1. Creational Patterns

Creational patterns focus on **object creation mechanisms**, making system independent of how objects are created, composed, and represented.

### Patterns:
- Factory Method
- Abstract Factory
- Builder
- Singleton
- Prototype

---

## 2. Structural Patterns

Structural patterns focus on **how classes and objects are composed** to form larger structures while keeping them flexible and efficient.

### Patterns:
- Adapter
- Bridge
- Composite
- Decorator
- Facade
- Flyweight
- Proxy

---

## 3. Behavioral Patterns

Behavioral patterns focus on **communication between objects**, defining how objects interact and distribute responsibility.

### Patterns:
- Strategy
- Observer
- Command
- Chain of Responsibility
- State
- Iterator
- Mediator
- Memento
- Template Method
- Visitor

---

## How Each Pattern Is Documented

Each pattern folder typically contains:

- `README.md` → Explanation (Intent, Motivation, UML, etc.)
- `implementation.cpp` → Code example (C++)
- UML diagram (if applicable)
- Violation vs Correct implementation (when relevant)
